About WIRM

The Web-based Interactive River Model (WIRM) is part of an on-going research project investigating advanced web-based approaches to environmental modeling and decision support. It is intended primarily as a demonstration of the use of modern web technologies to create a highly interactive, web-based interface for a water quality simulation model. This research is being conducted by Jeff Walker, PhD Candidate, and Dr. Steve Chapra, Professor in the Dept. of Civil and Environmental Engineering at Tufts University.

Purpose

Simulation models are widely used to support environmental management and decision making. However, most models require strong technical and programming skills due to antiquated or poorly designed user interfaces. This leaves decision makers and stakeholders reliant on modeling experts to develop, run and analyze models on their behalf.

The World Wide Web has evolved into a revolutionary software platform for creating interactive, easy-to-use, and widely accessible applications. For many years, environmental modelers have been interested in developing web-based models and decision support systems that would improve model accessibility and understanding by decision makers and stakeholders. Performance gains in JavaScript engines (e.g. Chrome's V8) as well as a diverse ecosystem of open-source JavaScript libraries (e.g. jQuery, Backbone, d3 to name a few) are capable of supporting client-based web applications for environmental simulation. WIRM demonstrates the use of these technologies to create a highly interactive, easy-to-use, and widely accessible application for water quality simulation.

Server Architecture

The WIRM server was built using the Django web framework, which is "a high-level Python Web framework that encourages rapid development and clean, pragmatic design." Django is a popular and mature framework that has been under active development for over many years.

In addition, this site utilizes a number of Django plug-ins including:

  • Django Rest Framework: for delivering a REST-based API for accessing model parameters, projects, users and comments (License)
  • Registration: user account registration and email-based activation (License)

Client Architecture

The client-side web application controls the user interface, simulation engine and visualization renderer, all of which are executed within the web browser. It is built using a number of free and open-source software libaries including:

  • jQuery: DOM manipulation, AJAX communication (License)
  • jQuery-ui: slider widgets (License)
  • Backbone.js: MVC application framework (License)
  • Twitter Bootstrap: front-end framework (License)
  • Numeric.js: element-wise array operations (License)
  • d3.js: data visualization (License)
  • Underscore.js: miscellaneous JavaScript utilities (License)
  • MathJax.js: equation rendering using Latex-style code (License)
  • Moment.js: date/time formatting (License)
